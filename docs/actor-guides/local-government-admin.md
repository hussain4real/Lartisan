# Local Government Admin Guide

Primary surface: `/lga`

## Purpose

Local Government Admins own day-to-day Lartisan operations inside one LGA. They bridge state oversight and field execution.

## Current Responsibilities

- View artisan profiles scoped to the assigned LGA.
- Review standard KYC submissions after field evidence is submitted.
- Approve, return, reject, or escalate KYC according to risk.
- Manage and review Area Agent territory assignments.
- Reassign territory coverage with a valid territory-assignment reason code.
- Moderate artisan profiles within LGA scope.
- Suspend artisan profiles when local evidence supports it.

## Standard Workflow

1. Sign in with an LGA Admin account.
2. Open `/lga`.
3. Review submitted KYC records.
4. Check attached documents, Area Agent visit notes, and local geography.
5. Approve standard-risk profiles with complete evidence.
6. Return incomplete profiles with clear correction notes.
7. Reject or escalate suspicious profiles using the right reason code.
8. Review Area Agent coverage and reassign territories when coverage changes.

## KYC Decision Rules

- Approve only when identity, location, and service evidence are consistent.
- Return when evidence is incomplete but correctable.
- Reject when identity, location, or legitimacy concerns are strong enough to block onboarding.
- Escalate when risk exceeds local authority or affects multiple LGAs.
- Every decision requires an appropriate KYC decision reason code.

## Territory Rules

- Area Agents should be assigned to specific active territories.
- Reassignments require a territory-assignment reason code.
- End old assignments instead of silently overwriting coverage history.

## Escalations Owned

- Standard KYC approval.
- Local territory coverage.
- Agent performance and local follow-up.
- Local suspensions, unless a state-level policy or risk issue is involved.

## Current Limitations

- Customer booking and dispute queues are planned later.
- Payout approval queues are planned later.
- LGA finance reporting is not yet a complete operations panel.
